# 🧁 Gourmet Revient — Simulateur de Coût de Revient Pâtissier

[![GitHub Pages](https://img.shields.io/badge/Deploy-GitHub%20Pages-blue?style=for-the-badge&logo=github)](https://julianperesson.github.io/Gourmetrevient/)

**Gourmet Revient** est un outil interactif professionnel conçu pour les pâtissiers (notamment dans le cadre du Chef d'Œuvre du CAP Pâtissier). Il permet de calculer précisément le coût de revient des créations, de simuler des marges et d'analyser l'impact du marché sur la rentabilité.

## 🚀 Fonctionnalités Clés

- **Carrousel 3D Interactif** : Une sélection visuelle et élégante des 10 grandes pâtisseries françaises.
- **Simulateur de Coût Complet** : Calcul incluant les matières premières, la main d'œuvre (MO), l'énergie et les charges fixes.
- **Simulation d'Impact Marché** : Analysez en temps réel l'impact d'une hausse des prix des matières premières ou d'une baisse de production sur votre marge.
- **Niveaux de Qualité Dynamiques** : Comparez instantanément les coûts entre les niveaux Industriel, Standard, Artisanal et Premium.
- **Fiche Technique Exportable** : Générez des rapports PDF professionnels incluant la fiche technique et les indicateurs financiers.
- **Mode Pédagogique** : Explications claires des termes financiers (Seuil de rentabilité, Marge brute, etc.).

## 🛠️ Technologies Utilisées

- **Frontend** : HTML5, CSS3 (Vanilla), JavaScript (ES6+).
- **Animations** : GSAP (GreenSock Animation Platform) pour les transitions et le carrousel.
- **Visualisation** : Chart.js pour les graphiques de rentabilité.
- **Export PDF** : html2pdf.js.
- **Stockage** : LocalStorage (pour les recettes personnalisées et la session utilisateur).

## 📦 Installation & Hébergement

Ce projet est entièrement statique, ce qui facilite son hébergement sur **GitHub Pages**.

### Déploiement sur GitHub Pages

1. Créez un nouveau dépôt sur GitHub.
2. Poussez tous les fichiers du projet sur la branche `main`.
3. Allez dans les **Settings** de votre dépôt.
4. Dans le menu de gauche, cliquez sur **Pages**.
5. Sous "Build and deployment", choisissez la source **"Deploy from a branch"** et sélectionnez la branche `main`.
6. Attendez quelques minutes et votre site sera accessible à l'URL fournie par GitHub.
